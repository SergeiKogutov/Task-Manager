## 📋 Техническое задание: Kanban-доска (Task Manager)

### 1. Цель проекта
Разработать одностраничное приложение (SPA) для управления задачами по методологии Kanban с использованием **React + TypeScript** (без внешних UI-библиотек). Данные хранятся в `localStorage`.

---

### 2. Функциональные требования

#### 2.1. Доска с колонками
- На доске должно быть **3 колонки** по умолчанию:
  - `"📋 To Do"`
  - `"🔄 In Progress"`
  - `"✅ Done"`
- Каждая колонка содержит список карточек (задач).
- Колонки отображаются горизонтально в ряд (flex-контейнер).

#### 2.2. Карточка задачи
Каждая карточка должна содержать:
- **Заголовок** (обязательное поле, текст до 100 символов).
- **Описание** (опционально, текст до 500 символов).
- **Приоритет** (выбор из: `низкий`, `средний`, `высокий`) — визуально выделяется цветом.
- **Дату создания** (генерируется автоматически при добавлении).
- **ID** (уникальный идентификатор, например, `crypto.randomUUID()`).

#### 2.3. Управление задачами
- **Добавление:** Кнопка "➕ Добавить задачу" над каждой колонкой. Открывается модальное окно с формой (заголовок, описание, приоритет).
- **Редактирование:** Клик по карточке открывает модалку с редактируемыми полями.
- **Удаление:** Кнопка "🗑️" на карточке (с подтверждением через `confirm`).
- **Перемещение:** Карточки можно перетаскивать между колонками (Drag-and-Drop).

#### 2.4. Фильтрация и поиск
- **Поиск:** Строка поиска, которая фильтрует карточки по заголовку во всех колонках.
- **Фильтр по приоритету:** Выпадающий список (`все`, `низкий`, `средний`, `высокий`).

#### 2.5. Сохранение состояния
- Все данные (колонки и карточки) сохраняются в `localStorage`.
- При обновлении страницы данные восстанавливаются.

#### 2.6. Сброс данных
- Кнопка "🔄 Сбросить доску" очищает `localStorage` и создает начальные 3 пустые колонки (с подтверждением).

#### 3.3. Типы (TypeScript)
```typescript
// types/index.ts

export type Priority = 'low' | 'medium' | 'high';

export interface Task {
  id: string;
  title: string;
  description?: string;
  priority: Priority;
  createdAt: Date;
}

export interface Column {
  id: string;
  title: string;
  tasks: Task[];
}

export type Board = Column[];

// Типы для пропсов компонентов
export interface ColumnProps {
  column: Column;
  onAddTask: (columnId: string, task: Omit<Task, 'id' | 'createdAt'>) => void;
  onDeleteTask: (columnId: string, taskId: string) => void;
  onEditTask: (columnId: string, taskId: string, updatedTask: Partial<Task>) => void;
  onMoveTask: (sourceColumnId: string, destColumnId: string, taskId: string) => void;
}

export interface TaskCardProps {
  task: Task;
  onDelete: () => void;
  onEdit: (updatedTask: Partial<Task>) => void;
  columnId: string;
}
```

### 4. Этапы разработки (по спринтам)

#### 🟢 Спринт 1: Базовая структура и состояния (Дни 1-2)
**Цель:** Настроить проект и отобразить статичную доску.

- [ ] Инициализировать проект через Vite: `npm create vite@latest kanban-ts -- --template react-ts`
- [ ] Создать интерфейсы `Task`, `Column`, `Board` в `types/index.ts`.
- [ ] Написать начальные данные (3 колонки с примерами задач) в `App.tsx`.
- [ ] Реализовать компоненты `Board`, `Column`, `Card` (без логики, только отображение).
- [ ] Добавить базовую стилизацию (flex-контейнер для колонок, карточки с тенями).

#### 🟡 Спринт 2: CRUD операции (Дни 3-4)
**Цель:** Добавить, редактировать и удалять задачи.

- [ ] Создать переиспользуемый компонент `Modal` (затемнение + контент внутри).
- [ ] Реализовать форму добавления задачи (заголовок, описание, приоритет) с валидацией (заголовок обязателен).
- [ ] Реализовать форму редактирования (все поля, включая изменение колонки? – пока пропускаем, только редактирование в той же колонке).
- [ ] Добавить кнопки удаления с `window.confirm`.
- [ ] Вынести логику в кастомный хук `useBoard` (хук должен возвращать `board`, `addTask`, `editTask`, `deleteTask`, `moveTask`).

#### 🟠 Спринт 3: Drag-and-Drop (Дни 5-6)
**Цель:** Реализовать перетаскивание карточек между колонками.

- [ ] Реализовать Drag-and-Drop с помощью **нативных HTML5 событий** (`draggable`, `onDragStart`, `onDragOver`, `onDrop`). **Важно:** Типизировать все события (`React.DragEvent<HTMLElement>`).
- [ ] Логика перемещения: удалить задачу из исходной колонки → вставить в целевую.
- [ ] Добавить визуальный фидбек при перетаскивании (подсветка колонки, изменение курсора).
- [ ] Обработать кейс: перетаскивание в ту же колонку (ничего не делать).

#### 🔵 Спринт 4: Фильтрация и поиск (Дни 7-8)
**Цель:** Добавить удобный поиск и фильтрацию.

- [ ] Создать компонент `SearchBar` (input с `onChange`).
- [ ] Создать компонент `PriorityFilter` (select с вариантами).
- [ ] Реализовать логику фильтрации: сначала применяется фильтр по приоритету, затем поиск по заголовку (регистронезависимый).
- [ ] Убедиться, что фильтры не изменяют исходные данные в `localStorage`.

#### 🟣 Спринт 5: Персистентность и финальная полировка (Дни 9-10)
**Цель:** Сохранять данные и провести рефакторинг.

- [ ] Написать кастомный хук `useLocalStorage<T>` (дженерик).
- [ ] Интегрировать хранение в `useBoard`: при каждом изменении доски сохранять в `localStorage`.
- [ ] Восстанавливать данные при загрузке (если пусто — использовать начальные данные).
- [ ] Добавить кнопку "Сбросить доску".
- [ ] Провести рефакторинг: вынести утилиты (генератор ID, функции сортировки) в отдельные файлы.
- [ ] Проверить, что проект собирается без ошибок (`npm run build`).
